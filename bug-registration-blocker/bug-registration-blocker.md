# БЛОКЕР: регистрация нового пользователя невозможна

**Приоритет:** Blocker (критический)  
**Тип:** Функциональный дефект  
**Статус:** Open  

---

## Описание проблемы
При заполнении всех полей формы регистрации валидными данными и нажатии кнопки "Register" пользователь **не создаётся** в системе. Войти с указанными данными невозможно.

Функционал регистрации **полностью сломан**.

---

## Где найдено
- **URL:** https://demoqa.com/register
- **Окружение:** Chrome 134 / Windows 10

---

## Шаги воспроизведения
1. Открыть страницу регистрации: https://demoqa.com/register
2. Заполнить **все** поля валидными данными:
   - First Name: `Test`
   - Last Name: `User`
   - UserName: `testuser2026`
   - Password: `Test@123456` (со спецсимволом)
3. Нажать кнопку **"Register"**
4. Попытаться войти с созданными данными на странице `/login`

---

## Фактический результат (Actual Result)
- После нажатия "Register" страница **перезагружается**
- Все поля **очищаются**
- Сообщение об успехе/ошибке **отсутствует** (или мелькает <1 секунды)
- Вход с данными `testuser2026` / `Test@123456` **невозможен**

---

## Ожидаемый результат (Expected Result)
- Пользователь успешно создаётся
- Появляется сообщение: *"Registration successful"*
- Происходит перенаправление на `/profile` или страницу логина
- Вход с созданными данными возможен

---

## Серьёзность (Severity)
**Blocker** — функционал полностью недоступен. Новые пользователи не могут зарегистрироваться.

## Приоритет (Priority)
**High** — требует немедленного исправления.

---

## Дополнительные проверки
| Пароль | Результат |
|--------|-----------|
| `Test@123456` (со спецсимволом) | ❌ Регистрация не удалась |
| `Pass123` (без спецсимвола) | ❌ Регистрация не удалась |
| `Test@1` (короткий) | ❌ Регистрация не удалась |

**Вывод:** Ни один пароль не приводит к успешной регистрации.

---

## Возможная причина
- Бэкенд API `/Account/v1/User` возвращает ошибку, которую фронтенд не обрабатывает
- Отсутствует или неверный endpoint для регистрации
- Проблема с валидацией на сервере
