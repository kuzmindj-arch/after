# Баг-репорт: при пустых обязательных полях отсутствует текстовое сообщение об ошибке

**Приоритет:** High (высокий)  
**Тип:** UI/UX / Доступность (Accessibility)  
**Статус:** Open  

---

## Описание проблемы
При отправке формы регистрации с пустыми обязательными полями браузер подсвечивает их красной рамкой, но **текстовое сообщение об ошибке отсутствует**. Пользователь видит красную рамку, но не понимает:
- Что именно не так
- Как исправить ошибку
- Какое поле требует внимания

---

## Где найдено
- **URL:** https://demoqa.com/register
- **Окружение:** Chrome 134 / Windows 10

---

## Шаги воспроизведения
1. Открыть страницу регистрации: https://demoqa.com/register
2. Оставить поле **First Name** пустым
3. Заполнить остальные поля валидными данными
4. Нажать кнопку **"Register"**

## Альтернативные шаги
Повторить для каждого обязательного поля:
- Last Name (пустое)
- UserName (пустое)
- Password (пустой)

---

## Фактический результат (Actual Result)
- Поле подсвечивается **красной рамкой** ✅
- **Текстовое сообщение** об ошибке ❌ отсутствует
- Нет подсказки, что нужно сделать

---

## Ожидаемый результат (Expected Result)
- Красная рамка + текстовое сообщение под полем или всплывающей подсказкой
- Пример сообщения: *"First Name is required"* или *"Please fill out this field"*

## Приоритет (Priority)
**High** — исправить до следующего релиза

---

## Соответствие стандартам доступности (WCAG)
Нарушается критерий:
- **3.3.1 Error Identification** — ошибка должна быть описана текстом
- **3.3.2 Labels or Instructions** — инструкции должны быть понятны

Пользователи с нарушениями зрения могут не заметить красную рамку. Текстовое сообщение необходимо для screen readers (экранных дикторов).

---

## Затронутые поля
| Поле | Красная рамка | Текстовое сообщение |
|------|--------------|---------------------|
| First Name | ✅ Есть | ❌ Нет |
| Last Name | ✅ Есть | ❌ Нет |
| UserName | ✅ Есть | ❌ Нет |
| Password | ✅ Есть | ❌ Нет |
