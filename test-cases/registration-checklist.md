# Чек-лист тестирования формы регистрации (demoqa.com/register)

## 📋 Общая информация
- **URL:** https://demoqa.com/register
- **Тип тестирования:** Функциональное, UI/UX, валидация
- **Приоритеты:** High (H), Medium (M), Low (L)

---

## 1. Позитивные сценарии (H)

| # | Проверка | Ожидаемый результат | Результат | Примечание |
|---|----------|---------------------|-----------|------------|
| 1.1 | Регистрация с валидными данными | Успешная регистрация, перенаправление на /profile |❌ |Сообщение появляется на <1 секунды, затем страница перезагружается, поля пустые. Регистрация не происходит |
| 1.2 | Использовать минимальную длину пароля (8 символов) | Регистрация успешна |❌ |Сообщение "Password must be at least 8 characters" мелькает и исчезает, поля очищаются |
| 1.3 | Использовать максимальную длину пароля (если есть ограничение) | Регистрация успешна |❌| Аналогично |
| 1.4 | Использовать цифры в First Name / Last Name | Зависит от требований (ожидается ошибка) |❌| Аналогично |

## 2. Негативные сценарии валидации (H)

| # | Проверка | Ожидаемый результат | Результат | Примечание |
|---|----------|---------------------|-----------|------------|
| 2.1 | Пустое поле First Name | Сообщение об ошибке "First Name is required" |❌ |Красная рамка ✅, текстового сообщения ❌ нет|
| 2.2 | Пустое поле Last Name | Сообщение об ошибке |❌ |То же поведение|
| 2.3 | Пустое поле UserName | Сообщение об ошибке |❌ |То же поведение|
| 2.4 | Пустое поле Password | Сообщение об ошибке |❌ |То же поведение|
| 2.5 | Пароль короче 8 символов (например, "Pass123") | Ошибка: "Password must be at least 8 characters" |❌ |Сообщение мелькает и исчезает, поля очищаются|
| 2.6 | Пароль без цифр | Ошибка: "Password must contain at least one digit" |⚠️ Не проверялось |Требование неясно, проверять после фикса бага|
| 2.7 | Пароль без заглавной буквы | Ошибка: "Password must contain at least one uppercase letter" |❌ ||
| 2.8 | Пароль без спецсимвола | Зависит от требований |❌ |Сообщение есть, но исчезает за <1 секунды. Поля очищаются|
| 2.9 | Имя пользователя уже существует | Ошибка: "User already exists" |❌ |Сообщение есть, но исчезает за <1 секунды. Поля очищаются|
| 2.10 | Поле First Name содержит спецсимволы (@, #, $) | Ошибка валидации |❌ |То же поведение|

## 4. Безопасность (M)

| # | Проверка | Ожидаемый результат |
|---|----------|---------------------|
| 4.1 | Пароль не передаётся в открытом виде (проверить в Network) | В запросе пароль должен быть зашифрован |
| 4.2 | Нет XSS-уязвимости (ввести `<script>alert(1)</script>` в имя) | Данные экранируются, скрипт не выполняется |
| 4.3 | Нет SQL-инъекции (ввести `' OR '1'='1` в UserName) | Ошибка валидации или экранирование |
