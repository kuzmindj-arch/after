# Баг-репорт: сообщения об ошибке исчезают за долю секунды (flash message)

**Приоритет:** Medium (средний)  
**Тип:** UI/UX  
**Статус:** Open  
---

**Скриншот проблемы:**
![Сообщение исчезает](./screenshots/screencho4.png)
---

## Описание проблемы
При отправке формы регистрации с некорректными данными (например, пароль без спецсимвола) появляется текстовое сообщение об ошибке, но оно **исчезает менее чем за 1 секунду**. Пользователь не успевает его прочитать.

---

## Где найдено
- **URL:** https://demoqa.com/register
- **Окружение:** Chrome 134 / Windows 10

---

## Шаги воспроизведения
1. Открыть страницу регистрации: https://demoqa.com/register
2. Заполнить все поля валидными данными
3. В поле Password ввести `Pass123` (без спецсимвола)
4. Нажать кнопку **"Register"**

---

## Фактический результат (Actual Result)
- Сообщение появляется на **< 1 секунды**
- Пользователь **не успевает прочитать** текст
- Сообщение исчезает
- Страница перезагружается
- Поля очищаются

## Текст сообщения (пойман с помощью замедления)
> *"Password must contain at least one non-alphanumeric character"*

---

## Ожидаемый результат (Expected Result)
- Сообщение остаётся на экране **минимум 3-5 секунд**
- ИЛИ сообщение остаётся до следующего действия пользователя
- Страница **не перезагружается**
- Поля **не очищаются** (пользователь исправляет только пароль)

## Затронутые сценарии
| Сценарий | Текст сообщения | Исчезает? |

| Пароль короткий | (то же сообщение) | ✅ Да |
| Пустые поля | (нет текстового сообщения) | — |

## Возможная причина
Слишком короткий таймер (setTimeout) перед перезагрузкой страницы:

```javascript
// Предполагаемый проблемный код
showErrorMessage("Password must contain...");
setTimeout(() => {
  location.reload(); // Перезагрузка стирает сообщение
}, 500); // 0.5 секунды — слишком мало
